# Contributing to Josephson Junction Fitting Toolkit

We welcome contributions to the Josephson Junction Fitting Toolkit! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or bugfix
4. Make your changes
5. Test your changes
6. Submit a pull request

## Development Setup

1. Install the package in development mode:
```bash
pip install -e .
```

2. Install development dependencies:
```bash
pip install -e .[dev]
```

3. Run tests:
```bash
python -m pytest tests/
```

## Code Style

- Follow PEP 8 style guidelines
- Use type hints where appropriate
- Write docstrings for all public functions and classes
- Keep line length under 88 characters

## Testing

- Write unit tests for new functionality
- Ensure all tests pass before submitting
- Aim for good test coverage
- Include integration tests for complex features

## Documentation

- Update README.md if adding new features
- Include docstrings with parameter and return type information
- Add examples for new functionality
- Update any relevant example scripts

## Submitting Changes

1. Ensure your code follows the style guidelines
2. Write clear commit messages
3. Include tests for new functionality
4. Update documentation as needed
5. Submit a pull request with a clear description

## Reporting Issues

- Use the GitHub issue tracker
- Include a clear description of the problem
- Provide a minimal example if possible
- Include relevant system information (OS, Python version, etc.)

## Feature Requests

- Use the GitHub issue tracker
- Describe the feature and its use case
- Explain why it would be beneficial
- Consider implementation complexity

Thank you for contributing!
